# 使用指南

[English](../en/user-guide.md) · [中文文档首页](../README.md)

## 认识工作台

OpenReel Studio 的核心不是一条隐藏的自动化脚本，而是三个可以互相配合的工作区：

- **聊天**：描述需求、补充约束、要求修改或让 Agent 执行多步任务。
- **创作画布**：查看和编辑最终会交付给你的文本、图片、视频和音频节点。
- **流程面板**：选择、搭建和保存可复用工作流，管理步骤、循环和依赖。

## 创建第一个项目

1. 第一次使用先完成 LLM 和所需媒体 Provider 配置。
2. 新建项目并填写能识别内容的名称。
3. 告诉 Agent 你要制作什么，包括主题、时长、画幅和风格。
4. 如果有参考图片、剧本或素材，直接上传或从资产库加入。
5. 对阻塞问题给出明确答案；非阻塞细节可以让 Agent 自行发挥。

一个有效的起始请求示例：

> 制作一支 15 秒、16:9 的国风奇幻短片。先做角色和场景参考图，再做四宫格分镜，确认后生成视频。整体冷色、电影光影，不要旁白。

## 第一次配置模型

OpenReel Studio 把模型分成两组：

- **LLM** 负责 Agent 对话、推理、提示词、评审和上下文压缩。
- **媒体 Provider** 负责真正生成图片、视频或音频。

打开右上角“设置”，至少配置一个启用的 LLM。需要生成哪类媒体，再配置对应的图片、视频或音频 Provider。

![视频 Provider 的普通前端配置](../assets/screenshots/model-config-video-provider.png)

普通配置顺序是：

1. 填写带版本的 API Base URL，例如 `https://api.example.test/v1`。
2. 填写服务商实际模型 ID 和 API Key。
3. 视频选择与服务商接口匹配的 UMA 协议和模型目标；图片或音频选择对应 Catalog 协议。
4. 保存并在节点模型下拉框确认配置可见。
5. 用最短提示词和默认参数做一次真实节点运行。

如果协议不在下拉框，不能只在前端随便选择一个近似协议。视频需要根据服务商 API 文档补充 UMA V2 协议和视频目标，图片/音频补充对应 Catalog。完整字段和排障方法见[模型配置与协议接入](./model-providers.md)。

模型、比例和分辨率由节点或工作流产物的前端设置传入；可复用 Workflow Spec 不保存 Provider 密钥，也不固定这些部署相关参数。

## 节点是什么

| 节点 | 常见内容 |
| --- | --- |
| 文本 | 剧本、分段、提示词、检查结果、制作说明。 |
| 图片 | 人物、场景、道具、分镜、首尾帧、风格参考。 |
| 视频 | 单个镜头、分段视频、拼接结果或最终成片。 |
| 音频 | 配音、音乐、音效或从视频分离的声音。 |

节点的 `parent` 只负责界面分组；真正的生产依赖由参考关系表达。改变上游图片不会自动偷偷改写下游结果，你可以决定何时重新运行。

## 参考图如何工作

参考图有两种不同用途：

1. **给 LLM 看**：当任务需要根据画面描述人物、场景或视频动作时，模型必须读取图片像素。
2. **给生成模型用**：图片只是图片或视频生成的输入，不需要先发送给 LLM。

在节点详情中检查参考角色是否正确。人物、场景和分镜可能同时影响视频节点，不要只连接最后一张分镜图而丢失一致性素材。

## 生成与历史

- 运行节点会使用当前提示词、模型和参考素材。
- 每次成功结果都会进入历史区。
- 可以从历史中选择旧结果作为当前预览和下游参考。
- 新一次生成失败时，节点保留最近一次成功结果，并在节点下方显示错误。
- “手动生成”表示流程可以先创建节点和提示词，在你确认后再调用媒体模型。

重试前先阅读节点错误。配置错误、内容策略错误和临时网络错误需要不同处理，不要连续盲目重试。

## 使用工作流

如果需求符合已有模板，选择模板、填写输入并运行即可。需要长期复用的新流程时，再进入工作流搭建模式。工作流运行后，真正的产物仍然落在创作画布，而不是藏在流程节点里。

详细操作见 [工作流指南](./workflows.md)。

## 图片查看与编辑

- 点击预览图可在屏幕范围内完整查看。
- 多次生成后在历史区选择当前要使用的版本。
- 图片编辑会产生候选结果，确认后再替换当前输出。
- 保存到资产库后，可以在其他节点或项目中重新引用。

## 视频剪辑

在视频节点上选择“剪辑”进入时间线。

基础流程：

1. 从媒体池把图片、视频或音频直接拖到目标轨道；拖动时先查看时间线预排列虚影。
2. 放到片段断层时自动吸附；空间不足时后方片段整体后移，不会切断已有片段。
3. 拖动播放头定位，使用分割工具把片段切成两个真实片段。
4. 拖动片段边缘调整入点和出点，不能超过源视频长度。
5. 移动片段完成排列，使用吸附对齐切点、播放头和标记。
6. 检查视频帧带和音频波形是否与源素材一致。
7. 调节片段音量、轨道音量、静音、淡入和淡出。
8. 选中视频片段后调整位置、缩放、旋转、不透明度和矩形裁剪。
9. 导出序列；完成的视频会作为新的画布视频节点出现。

把导入素材拖入时间线只建立剪辑序列关系，不会在创作画布上生成素材依赖线。

画面属性只作用于当前选中的视频片段。没有选中片段时，调整控件不会改变其他素材。

## 项目数据与恢复

- 刷新页面不会删除节点或媒体历史。
- `/clear` 只清理对话和运行上下文，不删除画布节点。
- 全量重置会删除当前项目的创作内容，执行前需要明确确认。
- 运行 trace 和诊断日志用于排障，不应作为项目交付内容公开。

## 推荐使用习惯

- 一个明确作品使用一个项目，避免把完全无关的视频混在同一上下文。
- 先确定故事和视觉基准，再批量生成昂贵的视频。
- 重要参考图保存到资产库，并使用清晰的节点标题。
- 失败后修正最小范围的节点，不要无理由重跑整条流程。
- 正式导出前检查画幅、时长、帧率、音量和素材授权。
