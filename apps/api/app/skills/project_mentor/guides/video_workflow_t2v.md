# 文生视频流程

适用：快速概念、没有参考图、不强求人物/场景严格一致。

文生视频不生成参考图片。固定文本骨架仍然要有：项目故事/剧本、人物、必要时的时间节拍和场景规划。
对应 `video` 节点写 `fields.production_path="text_to_video"`。

每段流程：

1. 写完整剧本/规划 `text` 节点：时间节拍、动作、对白、情绪、镜头节奏、地点、道具、前后衔接。
2. 完全根据文字写视频提示词：主体、场景、动作起止、镜头运动、时间节奏、风格、光线、连续性、负面约束、时长、画幅。
3. 创建并运行一个 `video` 节点；节点不依赖分镜图或参考图。

不要因为项目是视频就创建人物图、场景图、分镜图、首尾帧或故事模板图。T2V 的质量主要靠文字视频提示词。需要复用的提示词写法写到 skill；当前流程直接按 skill 和用户需求写 prompt。
