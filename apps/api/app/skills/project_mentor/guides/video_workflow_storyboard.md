# 分镜图生视频流程

适用：15秒段落有多个节拍、动作调度、连续性或镜头运动需要视觉控制。
一段默认一张宫格分镜图；不要一格一个图片节点，除非用户要求拆镜头或导出单镜头。
最终 `video` 节点写 `fields.production_path="image_to_video"`，并用
`fields.references` 指向已规划的分镜图或参考图 `image` 节点，图片引用使用 `role:"visual_reference"`。

每段流程：

1. 读取分段剧本和场景规划。
2. 普通成片先创建或复用人物参考图和场景参考图 `image` 节点，再创建宫格分镜图。双人打斗、角色互动、复杂动作和风格化视频都需要人物图与场景图作为视觉锚点。人物参考通常放在 `assets` 分组；本段主战场参考图通常放在对应 `segment` 下，让流程保持“人物 -> 分集/分段 -> 场景 -> 分镜 -> 视频”的顺序。
3. 直接写人物参考图、场景参考图和分镜图的 image prompt；不要用 `video_prompt` 或泛化“视频内容图片模板”写这些 image 节点。
4. 写一张宫格分镜图提示词：格子、节拍顺序、关键情节点、轴线、构图、人物/场景连续性、光线、风格、无字幕无水印。
5. 运行分镜图 `image` 节点。
6. 读取已完成的分镜图输出，并先看图或读取视觉分析；当前模型看不了图时明确说明看不了，不要假装看过。
7. 根据分段剧本和真实分镜图内容写该段 `video` 提示词：动作起止、镜头运动、时间节奏、连续性、时长、画幅、引用关系。
8. 为这一段创建并运行一个 `video` 节点。

分镜要求：剧情必须按段落因果连续推进；格子是关键 beat，不是随机好看的画面。构图要写景别、主体位置、前中后景、视线方向和人物站位；双人/多人戏保持180度轴线，左右关系和运动方向不能乱跳。关键帧选动作起势、转折、高潮、结果/钩子等重要节点，不选无信息的过渡姿势。

分镜图模式是图生视频路径：最终视频提示词必须等分镜图完成、看图或读取视觉分析后再写。
需要复用的提示词写法写到 skill；当前流程直接按 skill、段落故事和参考图写 prompt。
