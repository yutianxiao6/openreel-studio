# 单张分镜图生视频流程

适用：用户要求单独镜头、需要更高单图质量、关键镜头需要强控制，或宫格图信息太密影响画质。

特点：每个关键镜头/关键帧单独生成一张高质量图片，成本更高，但画面细节通常优于宫格分镜。
最终 `video` 节点写 `fields.production_path="image_to_video"`，并用
`fields.references` 指向对应单张分镜图 `image` 节点，图片引用使用 `role:"visual_reference"`。

每段流程：

1. 读取分段剧本和场景规划。
2. 规划本段关键镜头/关键帧列表；每个镜头写关键情节点、构图、角色、动作、场景、光线、风格和衔接。
3. 为每张分镜图创建独立 `image` 节点并写独立 prompt。
4. 运行这些图片节点并读取完成输出，并先看图或读取视觉分析；当前模型看不了图时明确说明看不了，不要假装看过。
5. 根据分段剧本和真实单张分镜图内容写该段 `video` 提示词。
6. 为这一段创建并运行一个 `video` 节点。

关键帧要求：只选影响剧情理解和视频运动的节点，例如动作起势、接触/碰撞、发现、反应、转折、高潮、落点或钩子；不要选重复姿势或空镜过渡。多张单图之间必须保持场景轴线、人物左右关系、视线方向、运动方向、服装和道具连续。每张图 prompt 都要写明确景别、构图、主体位置、前中后景、相机高度和光线。

单张分镜模式是图生视频路径：视频提示词等参考图完成、看图或读取视觉分析后再写。
需要复用的提示词写法写到 skill；当前流程直接按 skill 和节点目标写 prompt。
