# 场景图提示词 Skill

用于主场景 `image` 节点。场景图表达故事发生地点、空间结构、光线、材质和关键道具。输出写成纯文本 image prompt 正文，使用无人物场景四视图设定图写法。

## 写法

prompt 写成“无人物场景四视图设定图，2x2 四宫格，同一环境四个机位/角度”。四个画格共同表达同一地点。写入节点时直接生成正文，不保留模板占位符、字段说明、代码块或 JSON 包装。

- 全景建立镜头，展示空间规模、主体区域和风格基调。
- 入口或主方向视角，展示角色可能进入和行动的方向。
- 反向或侧向空间，展示空间连续性、遮挡和纵深。
- 道具细节或俯视布局，展示关键物件、路线和位置关系。

每个画格写清地点、空间结构、入口方向、行动路线、遮挡关系、关键道具、光线、材质、氛围和视觉风格。场景图作为环境参考，画面主体是环境和道具。

## 标准正文结构

```text
无人物场景四视图设定图，2x2 四宫格，同一环境四个机位/角度，电影美术设定稿质感。

地点与空间：
{地点名称、时代背景、空间规模、功能区域、入口方向、角色可能行动路线、遮挡和纵深关系}。

画格一，全景建立镜头：
{展示空间规模、主体区域、主要光源、色调和整体风格基调}。

画格二，入口或主方向视角：
{展示角色进入方向、主要动线、门窗/通道/楼梯/街口等结构关系}。

画格三，反向或侧向空间：
{展示空间连续性、遮挡关系、纵深层次、背景结构和可复用构图点}。

画格四，道具细节或俯视布局：
{关键道具、桌面/武器/装置/标识/路线图/俯视布局，明确位置和材质}。

光线与材质：
{主光源、辅光、阴影、天气/烟尘/霓虹/火光等环境动态，墙面、地面、金属、木材、布料、玻璃等材质}。

画面要求：
无人物主体，空间结构清楚，道具位置明确，四个画格属于同一地点，服务后续分镜和视频镜头复用。
```

## Standalone 节点推荐字段

以下字段只用于直接创建或更新 `image` 节点。Workflow Build Mode 只把用途、阶段和引用等业务字段写进 spec；模型、画幅、分辨率、宽高、画质和 fps 由前端运行配置提供。

- `fields.purpose`: `scene_reference`
- `fields.stage`: `visual_reference`
- `fields.aspect_ratio`: `16:9`
- `fields.resolution`: `2560x1440`
- `fields.quality`: `high`
- `fields.references`: 本段故事或总剧本用 `context`

## 检查点

- 四个画格属于同一个地点。
- 场景能支撑人物行动、分镜构图和视频镜头。
- 关键道具、光线和风格锚点清楚。
